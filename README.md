# My OS 2.0

개인 개발 환경을 **기억 의존형**에서 **상태 기반 실행 시스템**으로 전환하기 위한 README입니다. 

이 시스템의 목표는 다음과 같이 단순하고 명확합니다.

- 어디서든 **항상 같은 방식으로 시작**할 수 있도록 합니다.
- 지금 무엇을 해야 하는지 **애써 기억하지 않아도 되게** 합니다.
- 작업 흐름이 끊기더라도 **즉시 이어서 재개**할 수 있도록 돕습니다.
- 로컬 환경이 아닌 **Remote 중심**으로 상태를 안전하게 유지합니다.

---

## 소개

이 프로젝트는 단순한 dotfiles 모음이나 shell 설정 파일이 아닙니다. 

이 시스템은 저의 **Personal Development Operating System 2.0**입니다. 단순한 개발환경이라기 보다 일관되게 작업을 시작, 진행, 마무리하는 방식의 체계로 개발 측면에서 뇌가 동작하는 운영 체제에 가깝습니다.

---

## 제작 배경

개발 환경이 복잡해질수록 다음과 같은 문제들을 자주 겪게 됩니다.

- 어디서부터 다시 시작해야 하는지 잊어버립니다.
- 여러 대의 컴퓨터를 사용할 때 작업 상태가 끊어집니다.
- 이전에 하던 작업의 컨텍스트를 다시 떠올리는 데 많은 시간이 소요됩니다.
- 해야 할 일 자체보다 "정리되지 않은 상태"가 주는 심리적 부담이 더 큽니다.
- 결국 작업 시작 자체를 미루게 됩니다.

이 시스템은 이런 문제들을 최소화하고, 특히 다음과 같은 상황에서 생산성을 높일 수 있도록 설계되었습니다.

- 작업 기억(Working memory)에 대한 부담이 큰 경우
- 컨텍스트 전환(Context switching) 비용이 높은 경우
- 프로젝트 구조가 복잡해져서 시작이 막막해지는 경우
- 다양한 장치에서 동일한 흐름으로 작업하고 싶은 경우
- "생각"하는 시간보다 "즉각적인 실행"이 더 중요한 경우

## 핵심 철학

### 1. 기억에 의존하지 않습니다
필요한 정보를 머릿속이 아니라 **시스템에 기록하고 직접 볼 수 있도록** 합니다.

### 2. 시작점은 항상 동일합니다
터미널을 열 때 **진입점(Entry point)이 항상 같기 때문에 작업을 시작하기 쉬워집니다**.

### 3. Remote가 기준이 됩니다
로컬 환경은 단순한 작업 노드(Node)로 활용하고, 기준이 되는 상태는 항상 Remote에 보관합니다.

### 4. 행동이 명확하고 쉬워져야 합니다
좋은 시스템은 **다음 첫 걸음을 쉽게 내디딜 수 있게 도와줘야 합니다**.
