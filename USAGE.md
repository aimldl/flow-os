# 💻 사용 가이드 (Usage & Workflow)

이 문서는 My OS 2.0 환경에서 매일 어떻게 작업을 시작하고 종료하는지, 그 흐름을 설명합니다.

## 기본 작업 흐름 (Workflow)

권장하는 기본적인 작업 흐름은 다음과 같습니다.

```text
Start → TODO → RESUME → go → 작업 진행 → q
```

1. **시작**: 터미널 실행 및 최신 상태 동기화
2. **확인**: `TODO.md` 와 `RESUME.md` 확인
3. **재개**: `go` 명령어로 즉시 이전 작업 디렉토리 진입
4. **작업**: 현재 집중해야 할 일 실행 (헷갈릴 땐 다시 `RESUME.md` 확인)
5. **종료**: `q` 또는 `qq` 입력 → 상태 기록 → Push 후 터미널 종료

---

## 핵심 파일 안내 (Core Files)

시스템의 상태를 관리하는 파일들은 모두 `~` (홈 디렉토리)에 위치합니다.

- **`OS.md`**: 이 시스템이 어떻게 동작하는지 정의하는 문서입니다.
- **`TODO.md`**: 해야 할 작업 목록입니다. 한 줄에 하나의 작업을 실행 가능한 항목으로 간략히 기록합니다.
- **`RESUME.md`**: 다음에 어디서부터 다시 시작할지 기록해 두는 파일입니다. **가장 중요한 역할**을 담당합니다.

---

## 핵심 명령어 (Core Commands)

CLI 환경에서 즉각적인 작업 수행을 돕는 단축 명령어들입니다.

### 작업 재개 명령어
- `go`: `RESUME.md`에 기록된 1번째 항목의 디렉토리로 즉시 이동/실행합니다.
- `go2`, `go3`: 각각 2번째, 3번째 항목으로 이동합니다.

### 작업 종료 명령어
- `q` (표준 종료): 
  - 변경 사항 Push (`gpush`)
  - `RESUME.md` 상태 업데이트 프롬프트
  - 최종 검수 후 터미널 종료
- `qq` (빠른 종료): 
  - 검수 과정을 생략하고 상태 업데이트 후 즉시 종료
