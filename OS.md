# 💻 Flow OS > Workflow

🎯 목표: 기억하지 않고, 망설이지 않고, 바로 실행하는 상태 만들기

📌 목적: 기억 복원, 현 상태의 즉시 인지, 작업 재개의 마찰 최소화

👉 Flow OS는 
- '잡'생각을 흘려보내고 작업 흐름의 몰입 상태 (Flow State)로 들어가는데 도움이 됐으면 하는 바렘으로 설계한 (뇌의) 운영 체제
- '시작 → 유지 → 복귀 → 종료'를 하나의 흐름으로 연결한다

## 🔁 이상적인 하루 루틴 (메인 스트림)

터미널 시작 → Dashboard → go → 작업1 → 작업2 → 작업3 → clean → quit



## 🚀 기본 작업 흐름
```text
터미널 시작 → Dashboard → go → 작업1 ... 작업2 ... 작업3 ... → clean → quit
                               ├→ memo _↗ ├→ memo _↗ ├→ memo _↗
                               └→ todo _↗ └→ todo _↗ └→ todo _↗ 
```
- 작업 중 떠오르는 '잡'생각을 memo로 흘려보내고
- todo로 현재 상태를 즉시 파악한다 (기억 복원)
- 빠른 작업 재개로 실행 흐름으로 다시 진입한다

## 🧠 규칙

- ✅ 잡생각 = 무조건 memo에 기록
- ✅ 종료 전 반드시 정리 (clean)
- ✅ 항상 이어서 시작할 수 있는 상태를 유지

### 📴 다시 돌아갈 기회!
- 원치 않게 DMN 상태로 이동해서 '잡'생각에 빠졌더라도 주의 전환의 기회는 반드시 발생한다
  - 신체적 요구(예: 화장실, 배고픔)는 자연스럽게 발생하고, 
  - 외부 자극(예: 전화, 타인의 개입)은 예측없이 발생할 수 있다

- 이때 todo를 기준으로 현재 상태를 빠르게 복원하고, Flow OS의 메인 스트림으로 재진입하는 것이 중요하다

# 💻 Flow OS > Design

과학적 사실과 Flow OS 설계 방향을 설명합니다

## 🔬 과학적 사실 (요약): SN | CEN ↔ DMN
- CEN (Central Executive Network): 목표 지향 행동, 집중, 작업 수행을 담당  
- DMN (Default Mode Network): 자기 생각, 회상, 상상, mind-wandering 상태에서 활성  
- CEN ↔ DMN 관계: 한쪽이 활성화되면 다른 쪽이 억제되는 ‘반대 관계’ 구조  
- SN (Salience Network)의 역할: 중요한 자극을 감지하고, 네트워크 간 전환을 조절  

## 🚀 '잡'생각과 'job'생각에 관한 고찰
- 우선순위 상위 3개 작업, 즉 일(job)과 직접적으로 관련되지 않은 생각을 ‘잡’생각으로 정의한다
- ‘잡’생각은 쓸모없는 것이 아니라, 오히려 의미 있는 생각이기 때문에 기록할 가치가 있다
- 다만 ‘job’생각에 집중하기 위해서는, 중요한 ‘잡’생각일지라도 일시적으로 흘려보낼 필요가 있다
- 우리의 뇌는 외부 자극과 내부 상태에 반응하며 다양한 생각을 자동으로 생성하므로, ‘잡’생각 자체를 완전히 막는 것은 어렵다
- 따라서 핵심은 ‘잡’생각을 빠르게 외부에 기록(memo)하고, 현재의 작업 흐름으로 신속하게 복귀하는 것이다

## 🚀 'job'생각:'잡'생각 = CEN:DMN
일반적으로 집중 상태에서는 CEN이 활성화되고 DMN은 억제된다.
- ‘job’생각은 목표 지향적 작업과 실행 기능을 담당하는 CEN과 관련된다.
- ‘잡’생각은 내부 사고, 회상, 상상과 같은 활동과 관련된 DMN과 연결된다
- SN은 중요한 자극을 감지하고, 뇌가 내부/외부 상태 간 주의를 전환하도록 조절하는 역할을 한다

## ⚙️ Flow OS 설계 (아이디어 + 비유)

### 🧠 비유: 평야와 고원

- DMN은 자유롭게 이동하는 “넓은 평야”와 같다  
  → 이동이 쉽고 자연스럽지만, 방향 없이 떠다니기 쉽다  

- CEN은 집중을 요구하는 “고원(高原, plateau)”과 같다  
  → 진입이 더 어렵고 머무르는 것만으로도 더 에너지가 필요하다  

- 인간의 뇌는 기본적으로 평야(DMN)에 머무르기 쉽기 때문에  
  → 고원(CEN)으로 올라가는 경로를 명확하게 만드는 것이 중요하다  

### 🚀 설계 아이디어 
Flow OS는 
- 이런 뇌의 특성을 고려/수용하고, ‘잡’생각을 억제하기보다 빠르게 외부로 분리(memo)하고, 작업 상태(CEN)로 다시 복귀하도록 돕는 구조로 설계되었다

- 핵심 전략은 ‘잡’생각(DMN 활동)을 억제하는 것이 아니라,  
  이를 빠르게 외부로 분리(memo)하고 다시 작업 상태(CEN)로 복귀하는 것이다  

- 중요한 것은 “집중 유지”보다,
  “빠른 복귀”와 “낮은 재시작 비용”이다  

- 작업으로 돌아오는 과정에서 시작의 부담이 클수록  
  → 다시 내부 사고(DMN) 상태로 쉽게 이탈할 수 있다  

- Flow OS는 이 전환 비용을 줄여  
  → DMN 상태에서 CEN 상태로 보다 빠르고 안정적으로 복귀하도록 돕는다 

### 🧭 직관적 이미지: 설계 의도와 핵심 도구
이탈에는 가드레일을 치고, 이탈 시 재진입의 마찰을 최소화한다.

#### 🚧 memo

내부 사고(DMN)를 외부로 빠르게 배출하는 출구  

### 메모 명령어
- `memo [메시지]`: `flow-entry/temporary/memo.md`에 timestamp와 함께 메모를 저장합니다. 인자 없이 실행하면 대화식으로 입력받습니다.


#### todo
- 현재 위치와 목표를 명확히 보여주는 표지판
- 내가 어디에 있는지 인지하고, 다시 돌아갈 경로를 최소한의 노력으로 확보한다.
- 평야를 자유롭게 달리던 뇌가 언제든 고원으로 돌아갈 수 있도록  
  → 표지판(todo)과 진입 경로를 미리 만들어 둔다  

- 더 나아가,  
  → 복귀 경로를 “울퉁불퉁한 길”이 아니라  
  → “이미 닦여 있는 도로”로 만드는 것이 Flow OS의 역할이다

### 🚀 ADHD와 Flow OS

- ADHD에서는 작업 수행 중에도 DMN이 충분히 억제되지 않는 경향이 있으며,  
  이로 인해 '잡'생각인 내부 사고가 지속적으로 개입할 수 있다 (DMN intrusion) 
- SN은 중요한 자극을 감지하고 주의를 전환하는 데 관여하지만, ADHD에서는 이러한 전환의 효율이 낮아질 수 있다  
- CEN, DMN, SN 간의 조절과 전환이 원활하지 않아 집중 상태로의 전환 및 유지가 어려워질 수 있다
- 결과적으로 ADHD에서는  
  - → 자발적인 mind-wandering이 더 자주 발생하고  
  - → 작업 중 집중 유지가 더 많은 인지적 비용을 요구할 수 있다 
- Flow OS는 이러한 인지적 비용을 줄이고자 하는 바람에서 출발했다.
- ※ Disclaimer: 성인 ADHD를 기준으로 한 인지 흐름 모델이며, 아동 ADHD의 발달적 특성은 포함하지 않는다
