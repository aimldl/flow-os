# 작업 흐름 (Workflow)

## 기본 흐름

```
터미널 시작 → Dashboard → go → 작업 → clean → quit
```

1. **시작**: 터미널 실행 → 자동으로 git pull (스마트 sync) + Dashboard 표시
2. **확인**: todo.md와 resume.md 확인 (Dashboard에 자동 표시)
3. **재개**: `go` 명령어로 이전 작업 디렉토리에 즉시 진입
4. **작업**: 현재 집중해야 할 일 실행
5. **정리**: `clean` 으로 memo/todo 아카이브 (권장)
6. **종료**: `q` 또는 `qq` → 상태 기록 + git push + 터미널 종료

## 작업 중 이탈과 복귀

작업 중 '잡'생각이 떠오르면 `memo`로 흘려보내고, `todo`로 할일을 확인해서 'job'생각으로 돌아옵니다.

```
터미널 → Dashboard → go → 작업1 ... 작업2 ... 작업3 ... → clean → quit
                          ├→ memo ↗  ├→ memo ↗  ├→ memo ↗
                          └→ todo ↗  └→ todo ↗  └→ todo ↗
```

### memo: '잡'생각 배출

```bash
fmemo 떠오른 생각
```

- ❌ '잡'생각을 정리하는 곳이 아니다
- ✅ '잡'생각을 외부에 버리는 곳이다

### todo: 현재 위치 확인

- '잡'생각에 빠졌더라도 주의 전환의 기회는 반드시 발생한다 (신체적 요구, 외부 자극)
- 이때 `todo`를 기준으로 현재 상태를 빠르게 복원하고, 작업 흐름으로 재진입한다

## 세 가지 종료 방식 비교

```
터미널 시작 → 작업1 → 작업2 → 작업3                          (정리 없음)

터미널 시작 → Dashboard → go → 작업1 → 작업2 → 작업3 → quit  (자동 기록)

터미널 시작 → Dashboard → go → 작업1 → 작업2 → 작업3 → clean → quit  (권장)
```

일을 어떻게 마무리하느냐에 따라 다음 시작의 난이도가 크게 달라집니다.

## 종료 명령어

| 명령어 | 내부 동작 |
|--------|-----------|
| `q` (quit) | 네트워크 확인 → git push → resume 저장 → 검수 → 터미널 종료 |
| `qq` (quick-quit) | 네트워크 확인 → git push → resume 저장 → 즉시 터미널 종료 |

## 규칙

- ✅ 잡생각 = 무조건 memo에 기록
- ✅ 종료 전 반드시 q 명령어 실행 정리
- ✅ 항상 이어서 시작할 수 있는 상태를 유지
